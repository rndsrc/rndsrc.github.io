# A New Jupyter-Book Based Website

After using Wordpress and an NAS to host my personal website for 5
years, I finally decided to switch.
The main reason is that I evolved into a total git-geek---I want to
version control absolutely everything in git.
Wordpress' database content management is overkill for me.
Instead, all I need is a static website versioned by git and hosted by
GitHub.

In addition, as I use
[Jupyter Notebook](https://jupyter.org/)/[Lab](https://jupyterlab.readthedocs.io/en/stable/)
extensively to process
[EHT](http://eventhorizontelescope.org) data, I would love to
integrate Jupyter Notebooks into my website.
The [Jupyter-Book](https://jupyter.org/jupyter-book/intro.html)
project seems to be the perfect solution.
It supports both markdown (like [jekyll](https://jekyllrb.com/)) and
Jupyter Notebooks for content creation, and also support using
[Binder](https://mybinder.org/) to run notebooks live.
